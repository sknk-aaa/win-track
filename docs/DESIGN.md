# DESIGN — 勝率カウンター

## 目的

対戦ゲームなどの勝敗を、アプリ本体・ホーム画面ウィジェット・ロック画面ウィジェットから最短1タップで記録し、勝率をすぐ確認できるようにする。

## 対象

- iOSのみ。
- 最低OSはiOS 17以上を想定。
- Android、Web、海外向け展開は後回し。

## MVP

- 複数カウンター作成。
- カウンターごとの勝ち/負け記録。
- 勝率、勝ち数、負け数、合計試合数の表示。
- カウンターごとの名前、写真登録。
- 写真はライブラリ選択とカメラ撮影に対応。
- 履歴保存。
- 履歴ページで全カウンター横断表示。
- 履歴ページでカウンター絞り込み。
- 履歴から個別削除。
- アプリ内から直前の記録を取り消し。
- カウンターのアーカイブ、復元、完全削除。
- データ全削除。
- ホーム画面ウィジェットから勝ち/負け記録。
- ロック画面ウィジェットから勝ち/負け記録。
- アプリ内でウィジェット枠1〜3にカウンターを割り当て。
- 設定画面でアプリアイコンを選択。
- ライト/ダーク両対応。
- VoiceOver、Dynamic Type、Reduce Motion、44pt以上のタップ領域対応。

## 非対象

- 引き分け。
- ログイン。
- iCloud同期。
- サーバー保存。
- 課金。
- 広告。
- CSVエクスポート。
- グラフ。
- Live Activity。
- Control Center対応。
- ウィジェットへの写真表示。
- 履歴への記録元表示。
- ウィジェットからの取り消し。
- 連打対策。

## 計算式

- 勝率: `wins / (wins + losses)`
- 0試合時: `--%`
- 表示: 小数1桁。例: `62.5%`
- 履歴を正とし、削除や取り消し後は勝率・勝敗数・合計を即再計算する。

## データモデル

### Counter

- `id`
- `name`
- `photoUri`
- `isArchived`
- `createdAt`
- `updatedAt`
- `lastRecordedAt`

### MatchRecord

- `id`
- `counterId`
- `result`: `win` or `loss`
- `createdAt`

### WidgetSlot

- `id`: `slot1` / `slot2` / `slot3`
- `label`
- `counterId`

ウィジェットごとのカウンター選択は、アプリ内のウィジェット枠にカウンターを割り当て、ウィジェット側では枠を選ぶ方式にする。

### WidgetPendingEvent

- `id`
- `slotId`
- `counterId`
- `result`: `win` or `loss`
- `createdAt`

ウィジェットタップ時は、Swift WidgetKit側でウィジェット表示を即時更新し、App Group内の共有JSONとスナップショット内の `pendingEvents` に未同期イベントとして保持する。アプリ起動時/前面復帰時にSQLiteへ取り込み、取り込み後にウィジェットのスナップショットを再発行する。表示スナップショットは App Group のJSONファイルと `UserDefaults` の両方を読み、`updatedAt` が新しい方を使う。アプリ側のカスタムブリッジ保存が失敗した場合は `@bacons/apple-targets` の `ExtensionStorage` で `UserDefaults` 保存へフォールバックする。

## データ方針

- 端末内保存のみ。
- 写真は端末内保存のみ。サーバー送信しない。
- 写真は元画像を加工しない。表示時に正方形サムネとして見せる。
- アーカイブ済みカウンターは通常の一覧・履歴に表示しない。
- アーカイブ済みカウンターは設定画面から管理する。
- ウィジェット由来の記録もアプリ由来の記録と同じ履歴として扱う。

## ウィジェット

### 共通

- ウィジェットごとに対象カウンターを選択できる。
- 実装上は、ウィジェット設定で `枠1` / `枠2` / `枠3` を選び、アプリ内設定で各枠にカウンターを割り当てる。
- ウィジェットには写真を表示しない。
- 記録元はアプリ/ウィジェットで区別せず、同一の記録として扱う。
- SDK 54固定のため `expo-widgets` は使わず、Swift WidgetKit拡張で実装する。
- 記録後は App Group 共有のJSONファイルと `UserDefaults` のウィジェットスナップショットを即時更新し、WidgetKitの再読み込みを即時要求する。
- 即時反映はこのアプリの最重要体験。
- 参照中のカウンターがアーカイブ済みの場合は「カウンターなし」表示にし、記録ボタンを無効化する。

## UI方針

- シンプルで迷わないUI。
- 記録導線は常に勝ち/負けの2択。
- 勝ち=緑、負け=赤。
- ベースは白黒で、ライト/ダーク両対応。
- 詳細なレイアウト寸法は実装後に実機確認で調整する。
